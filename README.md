# SHI Chatbot Real - Quick Start Guide

This guide contains the exact sequence of commands and steps to start the application when you run it again.

---

## 🛠️ Step 1: Start External Services (Docker)
The backend requires **PostgreSQL**, **Redis**, and **Qdrant** to be active.

1. Ensure **Docker Desktop** is running.
2. In the project root (`d:\Projects\shi-chatbot-real`), run:
   ```powershell
   docker compose up -d
   ```

---

## 🦙 Step 2: Ensure Ollama is Running
The app relies on Ollama for LLM and text embeddings.
1. Make sure the **Ollama** desktop app is open and running.
2. If you haven't pulled the models yet:
   ```powershell
   ollama pull llama3.2:1b
   ollama pull nomic-embed-text
   ```

---

## 🐍 Step 3: Run the FastAPI Backend
1. Open a new terminal and run:
   ```powershell
   cd d:\Projects\shi-chatbot-real\backend
   ```
2. Activate the Python virtual environment:
   ```powershell
   .\venv\Scripts\Activate.ps1
   ```
3. *(Optional - Run if code updates or packages change)* Install dependencies and run migrations:
   ```powershell
   pip install -r requirements.txt
   alembic upgrade head
   ```
4. Start the backend:
   ```powershell
   python main.py
   ```
   * Access API Docs: [http://localhost:8000/docs](http://localhost:8000/docs)

---

## ⚡ Step 4: Run the Next.js Frontend
1. Open a new terminal and run:
   ```powershell
   cd d:\Projects\shi-chatbot-real\frontend
   ```
2. *(Optional - Run if package.json changes)* Install node modules:
   ```powershell
   npm install
   ```
3. Start the Next.js development server:
   ```powershell
   npm run dev
   ```
   * Access App: [http://localhost:3000](http://localhost:3000)

The entire Chatbot interaction with Ollama, including prompts, behavior, "thinking" filters, and vector search, is heavily concentrated in a few specific backend files.

Here is the breakdown of the files responsible for handling the Ollama Bot:

1. The Core RAG Pipeline & Prompt Construction
File: backend/routers/chatbot_routes.py This is the primary brain of the chat system. When a user sends a message, this file intercepts it and:

Constructs the System Prompt: It defines exactly how the bot should behave, setting up its persona and feeding it the context retrieved from the database.
Strips "Thinking Out Loud": It contains a regex function _strip_preamble() designed specifically to remove annoying LLM quirks (e.g., "According to the documentation...", "I see you are asking...") before returning the answer to the user.
Orchestrates the Search: It fetches FAQs via Postgres Full-Text Search and calls the Qdrant vector database for semantic similarity.
2. The Direct Ollama Communicator
File: backend/services/ollama_service.py This handles the actual HTTP network calls to your local Ollama instance (/api/chat and /api/embeddings).

LLM Settings: This is where hyperparameters for Ollama are defined (e.g., temperature, num_predict, top_p, repeat_penalty, seed).
Query Rewriting: It contains a specific prompt rewrite_query() that asks Ollama to rewrite follow-up questions using chat history so they make sense as a standalone database search.
Streaming: It handles the generate_stream() function that chunks the response so the frontend can type it out in real-time.
3. Intent Detection & Routing
File: backend/utils/intent_utils.py Before diving into RAG, the system likely checks the user's intent. This file uses Ollama to classify what the user is trying to do (e.g., is this a casual greeting, or do they need technical support?) to route the query properly.

4. The Vector Database (Context Retrieval)
File: backend/services/qdrant_service.py When the user asks a question, this file takes the embedding generated by Ollama and searches the Qdrant database to find the most mathematically similar FAQs and documents to inject into the Prompt in chatbot_routes.py.

Where should you look if you want to make changes?

To change how the bot answers/behaves (Persona/Prompt): Look at chatbot_routes.py
To change the LLM Temperature or Token Limits: Look at ollama_service.py
To change how strictly it matches similar FAQs: Look at the search limits and threshold scores in chatbot_routes.py and qdrant_service.py.
